# Design (설계)

기능 및 컴포넌트의 설계 문서를 관리합니다.

## 문서 형식

각 설계 문서는 다음 구조를 따릅니다:

```markdown
# [기능명] 설계

## 아키텍처
- 전체 구조
- 컴포넌트 다이어그램

## 컴포넌트 설계
- Props 인터페이스
- 상태 관리
- 이벤트 처리

## 스타일 설계
- 디자인 토큰 사용
- 레이아웃 구조
- 반응형 전략

## API 설계 (해당 시)
- 엔드포인트
- 요청/응답 스키마
- 에러 처리

## 의존성
- 사용할 라이브러리
- 다른 컴포넌트와의 관계
```

## 파일명 규칙

`YYYYMMDD-feature-name.md`

예: `20260128-button-component.md`

# PDCA Documentation Structure

이 폴더는 프로젝트의 PDCA(Plan-Do-Check-Act) 사이클을 관리합니다.

## 폴더 구조

```
docs/
├── 01-plan/      # 계획 문서
├── 02-design/    # 설계 문서
├── 03-do/        # 구현 기록
├── 04-check/     # 검증 및 분석
└── 05-act/       # 개선 및 학습
```

## 사용 방법

### 1. Plan (계획)
- 기능 요구사항 정의
- 목표 및 범위 설정
- 작업 계획 수립

### 2. Design (설계)
- 아키텍처 설계
- 컴포넌트 구조 설계
- API 설계

### 3. Do (구현)
- 구현 진행 상황 기록
- 주요 의사결정 기록

### 4. Check (검증)
- Gap 분석 결과
- 코드 리뷰 결과
- 테스트 결과

### 5. Act (개선)
- 개선 사항 정리
- 학습 내용 정리
- 최종 보고서

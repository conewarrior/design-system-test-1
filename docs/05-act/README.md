# Act (개선)

검증 결과를 바탕으로 한 개선 및 학습 문서를 관리합니다.

## 문서 형식

각 개선 문서는 다음 구조를 따릅니다:

```markdown
# [기능명] 개선 및 학습

## 개선 사항
- 수정된 이슈
- 적용된 최적화
- 리팩토링 내역

## 학습 내용
- 새로 배운 기술
- 효과적이었던 접근법
- 피해야 할 안티패턴

## 베스트 프랙티스
- 재사용 가능한 패턴
- 프로젝트 규칙 업데이트
- CLAUDE.md 반영 사항

## 다음 사이클 계획
- 추가 개선 필요 사항
- 다음 기능 개발 시 고려사항

## 최종 결과
- 달성한 목표
- 미달성 목표 및 이유
- 전체 사이클 평가
```

## 파일명 규칙

`YYYYMMDD-feature-name.md`

예: `20260128-button-component.md`

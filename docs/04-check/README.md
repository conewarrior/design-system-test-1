# Check (검증)

구현 결과의 검증 및 분석 문서를 관리합니다.

## 문서 형식

각 검증 문서는 다음 구조를 따릅니다:

```markdown
# [기능명] 검증

## Gap 분석
- 설계 vs 구현 비교
- 누락된 기능
- 추가된 기능
- 불일치 사항

## 코드 리뷰
- 코드 품질 평가
- 발견된 이슈
- 개선 제안

## 디자인 시스템 준수
- 토큰 사용 검증
- design-rules 준수 확인
- 위반 사항 및 수정 필요 항목

## 테스트 결과
- 테스트 커버리지
- 실패한 테스트
- 버그 리포트

## 성능 분석
- 렌더링 성능
- 번들 크기
- 최적화 필요 사항
```

## 파일명 규칙

`YYYYMMDD-feature-name.md`

예: `20260128-button-component.md`

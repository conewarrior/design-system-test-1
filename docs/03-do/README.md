# Do (구현)

구현 과정의 기록과 주요 의사결정을 관리합니다.

## 문서 형식

각 구현 기록 문서는 다음 구조를 따릅니다:

```markdown
# [기능명] 구현 기록

## 구현 일정
- 시작일
- 완료일

## 구현 내용
- 구현된 기능
- 추가된 파일 목록
- 주요 코드 변경 사항

## 의사결정
- 설계 변경 사항
- 기술 선택 이유
- 트레이드오프

## 이슈 및 해결
- 발생한 문제
- 해결 방법
- 참고 자료

## 테스트
- 수동 테스트 결과
- 확인된 시나리오
```

## 파일명 규칙

`YYYYMMDD-feature-name.md`

예: `20260128-button-component.md`
